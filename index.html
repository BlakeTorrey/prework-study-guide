HTML

  • The head element contains information about the webpage.
  • The body element represents visible content shown to the user.
  • The h(1-6) element represents headers, use h1 only once per page.
  • Use the u1 and o1 parent elements for lists, use the child element li for the content in the lists.
  • HTML is the foundation for wepages, make sure you understand its elements well!
    • If the spacing isn't working, adding p elements and the child br element breaks lines up.

    CSS

  • The * selector applies its rules to all elements in the page.
  • Margin and padding add buffer space around elements, margin around the page and padding in the element itself.
  • There are three styles of CSS formating; Inline CSS, Internal CSS Style Sheet, and External CSS Style sheet.
  • External CSS style sheets must be linked to the main html code in the header section.
  • Adding a . in front of something (card) creates a class (card) and any selector can be assigned to the class (card).

    Git

  • git status: Checks what branch you are currently on!
  • git checkout -b (branch-name): creates a new branch and switches to it.
  • git pull origin main: checks to make sure the branch is up to date.
  • git add -A: adds all files to commit.
  • git commit -m "(comments)": commits branch with comments on changes.
  • git push origin "name of branch": pushes branch to a pull request on the main.
    •

    Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control Flow is the order in which the computer exectues the code.
  • A function lets us store a series of operations under a variable name.
  • A for loop allows us to rerun the same operation without needing to duplicate the instructions.
